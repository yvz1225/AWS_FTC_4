# 팀플 AI Agent - 요구사항 명세서 (MVP)

## 1. 서비스 개요

**팀플 AI Agent**는 팀 프로젝트 진행 과정에서 역할 분담, 일정 조율, 업무 추적, 활동 모니터링, 리마인드 메일 발송을 지원하는 협업 보조 서비스입니다.

### 해결하고자 하는 문제

- 팀원별 역할 분담이 애매한 문제
- 해야 할 일을 체계적으로 정리하기 어려운 문제
- 누가 얼마나 참여하고 있는지 파악하기 어려운 문제
- 마감 직전까지 업무가 방치되는 문제
- 팀 프로젝트 운영 툴이 흩어져 있는 문제

---

## 2. 핵심 기능

### 기능 1. when2meet 스타일 가용시간 입력 UI

**목표:** 팀원들이 각자 가능한 시간을 직접 표시해서 공통 회의 가능 시간을 빠르게 확인할 수 있도록 합니다.

**구현 방향:** 외부 오픈소스를 깊게 붙이지 않고, when2meet 스타일의 간단한 시간표 UI를 직접 구현합니다.

**주요 기능:**
- 주간 단위 시간표 표시
- 요일 × 시간대 그리드 제공
- 팀원이 가능한 시간을 클릭/드래그로 선택
- 여러 팀원의 선택 결과를 합쳐서 공통 가능 시간 시각화
- 팀별 회의 후보 시간 확인

**저장 데이터:**

| 필드 | 설명 |
|------|------|
| project_id | 프로젝트 식별자 |
| member_id | 팀원 식별자 |
| day_of_week | 요일 |
| start_time | 시작 시간 |
| end_time | 종료 시간 |
| availability_status | 가용 상태 |

**기대 효과:**
- 회의 시간 조율용 외부 서비스 이동 없이 한 플랫폼 내 해결
- 팀 프로젝트 초반 일정 정리 효율화

---

### 기능 2. 팀원별 활동 지표 확인

**목표:** 팀원별 협업 참여 정도를 정량적으로 참고할 수 있도록 합니다.

**활동 지표 산정 기준:**
- Notion과 GitHub 체크리스트 중 둘 다 선택하거나 하나만 선택 가능

**Notion 기반:**
- 체크리스트 완료 개수
- 마감 준수 여부

**GitHub 기반:**
- 커밋 개수

**표시 방식:**
- 가중치 기반으로 활동 지표를 계산해서 대시보드에 표시

```
Notion Activity = 완료 개수 + 정시 완료 보너스
Git Activity    = 커밋 개수 + 마감 전 활동 여부
Activity Estimate = Notion * 0.5 + Git * 0.5
```

> ⚠️ 이 수치는 절대적인 평가값이 아니라, 협업 상태를 참고하기 위한 활동 지표로 사용합니다.

**대시보드 표시 항목:**
- 팀원별 완료 task 수
- 팀원별 최근 커밋 수
- 팀원별 마감 준수율
- 활동 상태 배너: 기여도 낮음 / 주의 / 정상 (퍼센트 표시)

---

### 기능 3. Task별 Deadline + 자동 리마인드 메일 발송

**목표:** 마감 직전까지 미완료 task가 방치되지 않도록 자동 리마인드를 보냅니다.

**방식:** 1시간마다 검사하는 스케줄러 방식

**동작 흐름:**
1. AWS Lambda가 매시각 실행
2. 전체 task 중 **내일 마감 + 아직 미완료(Status ≠ Done)** 인 항목 조회
3. 현재 시간이 12:00이면 해당 task 담당자에게 자동 메일 발송

**메일 발송 조건:**
- 마감일 하루 전
- 한국 시간 기준 12:00
- task 상태가 아직 완료가 아닐 것

**메일 내용:**
- 프로젝트명
- 담당 task 이름
- 마감일
- 현재 상태
- 리마인드 메시지

**기술 구조:**
- EventBridge 또는 스케줄 트리거
- AWS Lambda
- Notion DB 조회
- AWS SES 이메일 발송

**장점:**
- task 생성 시마다 개별 예약을 만들 필요 없음
- 상태 변경만 반영되면 전체 로직이 단순함
- 디버깅과 운영이 쉬움

---

### 기능 4. Gemini 기반 채팅형 명세 생성 + 승인 후 Notion 자동 생성

**목표:** 팀장이 채팅으로 프로젝트 정보를 입력하면, AI가 역할 분담과 할 일을 정리하고, 최종 승인 시 Notion에 자동으로 협업 구조를 생성합니다.

**채팅창에서 수집하는 정보:**
- 프로젝트명
- 팀원 이름
- 팀원 역할
- GitHub ID
- 이메일
- 저장소(repo) 주소
- 마감일
- 필요한 작업 범위

**채팅형 동작 흐름:**
1. 사용자가 프로젝트 정보를 채팅창에 입력
2. Gemini가 대화를 바탕으로 역할별 task 초안 생성
3. 사용자가 수정 요청
4. Gemini가 수정 반영
5. 이 과정을 반복하며 명세 구체화
6. 사용자가 **승인 버튼** 클릭
7. 백엔드가 최종 명세를 검증
8. 검증 완료 후 Notion API 호출
9. 프로젝트 페이지 + 체크리스트 DB 자동 생성

**승인 후 생성되는 Notion 구조:**

1. **프로젝트 메인 페이지**
   - 프로젝트명
   - repo URL
   - 전체 진행 현황

2. **공통 Task DB**
   | 속성 | 설명 |
   |------|------|
   | Task | 업무명 |
   | Assignee | 담당자 |
   | Deadline | 마감일 |
   | Status | Todo / Doing / Done |
   | GitHubID | GitHub 계정 |
   | Email | 이메일 |
   | Category | 분류 |

3. **팀원별 페이지 또는 필터 뷰**
   - 해당 팀원의 task만 보이도록 필터링
   - 개인 진행 현황 확인 가능

---

### 기능 5. 재시도 버튼으로 새 채팅 시작

**목표:** 명세를 처음부터 다시 작성하고 싶을 때 빠르게 초기화합니다.

**동작:**
1. 새 `conversation_id` 생성
2. 이전 draft/spec 폐기 또는 별도 보관
3. UI 상태 초기화
4. 새 채팅 세션 시작

**결과:**
- 이전 수정 내역에 끌려가지 않음
- 완전히 새로운 프로젝트 초안 생성 가능

## 3. Notion 연동 전략

### 핵심 원칙

Notion 자동화를 너무 많이 기대하지 않는다. Notion은 체크리스트/페이지 생성에는 적합하지만, 버튼 자동화와 외부 로직 연결까지 전부 API만으로 해결하려고 하면 복잡해진다.

### MVP 전략

- 체크리스트 자동 생성까지만 Notion API 사용
- 완료 여부는 **Status 속성**으로 관리
- 백엔드가 주기적으로 Notion DB를 조회해서 상태 수집
- **버튼 기반 완료 보고는 사용하지 않음** → Status = Done이면 완료로 판정
- 마감 리마인드 / 활동 집계는 백엔드가 담당

### 이 방식의 장점

- Notion 플랜 제약 영향이 작음
- 구현이 단순함
- 디버깅이 쉬움
- Notion 버튼 자동화 의존도를 줄일 수 있음

## 4. 백엔드 검증 로직

LLM이 만든 명세를 그대로 저장하면 위험하므로, 반드시 백엔드에서 스키마 검증을 수행해야 합니다.

### 검증 항목

| 항목 | 설명 |
|------|------|
| 이메일 형식 검증 | 유효한 이메일 포맷인지 확인 |
| GitHub ID 형식 검증 | 유효한 GitHub 사용자명인지 확인 |
| 중복 팀원 제거 | 동일 팀원 중복 입력 방지 |
| deadline 파싱 실패 처리 | 날짜 형식 오류 대응 |
| 누락 필드 보완 | 필수 필드 누락 시 기본값 또는 에러 |
| 역할 값 정규화 | 역할명 통일 처리 |
| 빈 task 제거 | 내용 없는 task 필터링 |
| repo URL 형식 검증 | 유효한 GitHub repo URL인지 확인 |

> Gemini는 초안을 잘 만들 수 있지만, 입력 형식 오류나 누락을 완전히 책임질 수는 없기 때문입니다.

## 5. 시스템 전체 흐름

```
Step 1. 팀장 입력
  → 채팅창에서 프로젝트 정보 입력

Step 2. Gemini 초안 생성
  → 역할 분담, task 목록, deadline 구조화

Step 3. 사용자 수정
  → 채팅으로 계속 수정/보완

Step 4. 승인
  → 사용자가 승인 버튼 클릭

Step 5. 백엔드 검증
  → 입력값 검증, 스키마 정리, 누락 필드 보완

Step 6. Notion 생성
  → 프로젝트 페이지 생성
  → 공통 체크리스트 DB 생성
  → 팀원별 뷰/페이지 생성

Step 7. 주기적 모니터링
  → Lambda가 1시간마다 Notion/GitHub 조회
  → 활동 지표 계산
  → 마감 리마인드 여부 판정

Step 8. 자동 메일 발송
  → 내일 마감 + 미완료 task 대상
  → 정오 12시 자동 발송

Step 9. 대시보드 표시
  → 팀원별 활동 지표
  → 미완료 업무
  → 마감 임박 경고
  → 최근 GitHub 활동
```

## 6. UI 구성 (화면 설계)

첨부된 웹 페이지 디자인 기준으로 다음과 같은 레이아웃을 구성합니다.

### 메인 페이지 레이아웃: "Talk to Notion-Build"

> 채팅으로 요구사항을 정리하고 바로 실행 가능한 계획으로 바꿉니다.

#### 좌측 패널 - 요구사항 채팅

- 채팅 인터페이스 (말풍선 형태)
- 사용자 입력 → AI 응답 반복
- 하단 입력창 + 전송 버튼
- **retry** 버튼: 새 채팅 세션 시작
- **config** 버튼: 설정 패널 열기

#### 우측 상단 - 프로젝트 요약

- AI가 정리한 프로젝트 요약 텍스트
- **Markdown 다운로드** 버튼

#### 우측 중단 - 전체 진행률

- 프로그레스 바 (퍼센트 표시)
- 전체 task 완료율 시각화

#### 우측 중하단 좌 - 팀원 참여도

- 팀원별 원형 차트 (도넛 그래프)
- 퍼센트 수치 표시
- 상태 라벨: 기여도 낮음 / 정상 / 주의
- 경고 배너: "경고 N명" 표시

#### 우측 중하단 우 - 마일스톤

- 주요 마일스톤 목록
- 각 항목: 마일스톤명 + 마감일 + 상태(완료/진행 중/대기)

#### 상단 알림 배너 - 무임승차 경고

- 참여도가 낮은 팀원 경고 팝업
- 팀원명 + 참여도 퍼센트 + 이메일
- **메일 보내기** 버튼

#### 추가 예정 - 시간표 UI

- when2meet 스타일 가용시간 입력 그리드
- 요일 × 시간대 매트릭스
- 클릭/드래그로 시간 선택
- 공통 가능 시간 하이라이트

## 8. MVP 핵심 포인트

이 프로젝트의 MVP는 다음 세 가지를 가장 강하게 보여주면 됩니다.

### 1) 채팅 기반 분업 명세 생성
- 팀장이 자연어로 입력
- AI가 역할과 task를 구조화

### 2) 승인 후 Notion 체크리스트 자동 생성
- 협업 인프라를 즉시 세팅

### 3) 마감 기반 자동 리마인드 + 활동 지표 대시보드
- 실제 운영 보조 기능 제공

## 9. 최종 정리

- 시간표 기능은 when2meet 스타일의 간단한 직접 구현 UI로 제공한다.
- Notion은 체크리스트 생성 및 상태 관리용 협업 보드로 사용한다.
- 완료 처리는 버튼이 아니라 **Status 속성 변경**으로 처리한다.
- 활동 평가는 기여도 점수가 아니라 **활동 지표 / 기여도 추정치**로 표현한다.
- 마감 하루 전 정오에 자동 메일을 발송하기 위해 **1시간마다 실행되는 Lambda 기반 검사 구조**를 사용한다.
- 채팅 기반 명세 생성은 Gemini가 담당하되, 최종 저장 전에는 반드시 **백엔드 스키마 검증**을 거친다.
- 재시도 버튼은 새 `conversation_id`를 발급하여 완전히 새로운 채팅 세션을 시작한다.
